# tar2parquet 제약사항

## 1. 입력 파일

- 입력 파일은 `A.tar.gz` 형식이다.
- `A.tar.gz` 내부에는 여러 개의 CSV 파일이 존재한다.
- CSV 파일명은 다음과 같은 패턴이다.

```text
A-1.csv
A-2.csv
A-3.csv
...
```

- CSV 파일은 동일한 데이터 집합을 구성한다.
- 각 CSV 파일에는 header가 포함되어 있다.
- CSV 파일들의 header는 동일하다.

---

## 2. 데이터 규모

- `A.tar.gz` 파일 크기는 일반적으로 **1~2GB**이다.
- 압축 해제된 CSV 전체 크기는 일반적으로 **3~5GB**이다.

---

## 3. 출력 파일

- `A.tar.gz`는 `A.parquet`으로 변환되어야 한다.
- 여러 CSV 파일의 데이터는 하나의 Parquet 파일로 통합되어야 한다.

---

## 4. 디스크 사용 제약

- 압축 해제된 CSV 파일을 디스크에 저장하지 않는다.
- 중간 CSV staging 파일을 생성하지 않는다.
- 변환 과정에서 CSV 전체 크기(3~5GB)에 해당하는 추가 임시 디스크 공간을 요구하지 않는다.

---

## 5. 처리 방식 제약

- `A.tar.gz`를 **순차적으로 처리한다.**
- archive를 파일별로 반복해서 재탐색하지 않는다.
- 입력 archive를 가능한 한 **1회만 읽는다.**
- 전체 archive 또는 전체 CSV 데이터를 메모리에 적재하지 않는다.
- streaming 방식으로 처리한다.

---

## 6. 성능 제약

- **최대한의 처리 성능(throughput)을 목표로 한다.**
- 불필요한 중간 프로세스 및 데이터 복사를 최소화한다.
- 데이터 처리 과정에서 불필요한 CSV 재파싱을 수행하지 않는다.
- 성능에 불리한 row-by-row 처리 방식은 지양한다.

---

## 7. 구현 언어 및 실행 방식

- 구현 언어는 **Go**이다.
- 단일 Go 프로그램으로 실행 가능해야 한다.
- `A.tar.gz`를 입력받아 `A.parquet`을 생성하는 CLI 프로그램이어야 한다.

예시:

```bash
tar2parquet A.tar.gz
```

---

## 8. 외부 명령어 제약

다음 외부 명령어를 사용하지 않는다.

- `tar`
- `gzip`
- `awk`
- `sed`
- `cat`

---

## 9. CSV Header 제약

- 첫 번째 CSV의 header만 유지한다.
- 두 번째 이후 CSV의 header는 최종 데이터에 포함되지 않아야 한다.
- CSV header 처리를 위해 archive를 다시 읽지 않는다.
- header 제거를 위해 `awk`, `sed` 등의 외부 명령어를 사용하지 않는다.

---

## 10. 메모리 제약

- 전체 `tar.gz` 파일을 메모리에 적재하지 않는다.
- 전체 CSV 파일을 메모리에 적재하지 않는다.
- 전체 row를 메모리에 적재하지 않는다.
- 고정 크기 buffer 또는 streaming 기반 처리만 허용한다.

---

## 11. 오류 처리 제약

다음 오류는 반드시 감지되어야 한다.

- 입력 파일 open 실패
- gzip 처리 오류
- tar 처리 오류
- CSV 데이터 읽기 오류
- Parquet 생성 오류

오류 발생 시:

- 변환은 실패 처리한다.
- 오류 내용을 확인할 수 있어야 한다.
- 오류를 무시하고 정상 종료해서는 안 된다.

---

## 12. 입력 데이터 전제

- CSV 파일들은 동일한 schema를 가진다.
- CSV header는 각 파일의 첫 번째 line에 존재한다.
- CSV 파일 내부의 데이터는 정상적인 CSV 형식이라고 가정한다.
- CSV 파일 간 schema 변경은 고려하지 않는다.

---

## 13. 핵심 제약 요약

> **Go 단일 프로그램으로 `A.tar.gz`를 순차 streaming 처리하고, 압축 해제 CSV를 디스크에 저장하지 않으며, 외부 shell 명령어를 사용하지 않고, 전체 데이터를 메모리에 적재하지 않은 상태에서 최대 처리 성능으로 `A.parquet`을 생성해야 한다.
